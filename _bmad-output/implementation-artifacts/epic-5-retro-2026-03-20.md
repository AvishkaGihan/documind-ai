## Successes & Strengths
- **Clean Streaming Transport & Architecture:** Successfully implemented SSE streaming answer endpoint using `dio` stream transport with no new bloated dependencies.
- **Robust RAG Orchestration:** Seamlessly integrated semantic retrieval with LLM answer generation and deterministic page-level citations without LLM hallucination.
- **Strict Context Isolation:** Clean contextual boundaries applied between document switching and new conversation sessions, maintaining safe and isolated conversation memory.
- **Bootstrap Simplification:** Simplified the mobile client integration by adding a `/latest/messages` bootstrap endpoint, removing complex conversation-ID tracking from the client.

## Challenges & Growth Areas
- **Test Harness Constraints (Again):** Chat UI streaming widget tests triggered timeouts using `pumpAndSettle()` around animations and the text field. Continued enforcement of bounded `pump(Duration)` interval frames is necessary.
- **Message Filtering Complexity:** Ensuring data isolation during conversation message retrieval required explicit table joins, as `messages` don't possess a `user_id` column natively.

## Key Insights & Learnings
- **Deterministic Token Tracking:** Extracting citations deterministically via regex over the streamed response is significantly safer and more reliable than trusting the LLM to format JSON citations.
- **Stateless Client Preference:** Shifting active conversation resolution to the backend (latest updated) dramatically simplified Flutter client state management.
