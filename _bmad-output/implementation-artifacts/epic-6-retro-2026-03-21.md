## Successes & Strengths
- **Centralized UI Primitive Helpers:** Successfully introduced shared components like `LoadingShimmer`, `ProcessingAnimation`, and `app_snackbar.dart` early. This avoided duplicate layout decisions and accelerated late-stage stories (6.3, 6.4, 6.5).
- **Graceful Offline Fallbacks:** Designed a robust namespaced local cache utilizing Hive. Uploads and questions queue flushing triggers flawlessly upon connectivity restoration without impacting visual smoothness.
- **Centralized Error and Rate Cooldown Mapping:** Achieved excellent separation of concerns by integrating backend custom 429 headers directly into stateful rate-limit cooldown timers in the Chat controller.
- **Sentence-Buffered Live Announcements:** Implemented buffered semantics announcements that accumulate tokens sentence-by-sentence during streaming responses, delivering clean VO/TalkBack pacing.

## Challenges & Growth Areas
- **Material 3 Interaction Constraints in Tests:** Minor visual flakiness surfaced when asserting stream buffers due to default Flutter Material 3 `InkRipple` tap responses triggering timer delays. Handled with static theme configuration overrides inside widget test scopes.
- **Over-Gating Animations on Tests:** Verify bounded pumps (`pump(Duration)`) are firmly enforced for streamers or response bubbles to keep test runtimes lean without timing out.

## Key Insights & Learnings
- **Pre-emptive Token/Breakpoint Guards:** Structuring `responsive_breakpoints.dart` and `MediaQuery.disableAnimations` globally prevents manual toggling overrides inside nested child layouts.
- **Backing Abstractions with Fakes for Tests:** Creating explicit state machines/fakes for connectivity and rate cooldown trackers simplified visual regression assertions dramatically.
