## Successes & Strengths
- **Standardized Error Handling:** Establishing `build_error_detail` and global normalization helpers on the backend simplified endpoint creation and guaranteed client-side safety.
- **Clean Invalidation Logic:** Choosing an invalidate-by-time strategy (`User.token_invalid_before`) kept the setup stateless and maintainable for MVP scope.
- **Polished UX and Persisted Sessions:** The Flutter implementation accurately consumes backend tokens using `flutter_secure_storage`, manages multiple redirect states cleanly with Riverpod, and uses correct layout tokens and validation feedback.

## Challenges & Growth Areas
- **Form/Route Flash Prevention:** Designing route paths so `AsyncLoading` states and startup bootstrap routes never flash the login wall on warm boots.
- **Dependency Version Compatibility:** Averting breaking compatibility pins with `bcrypt` / `passlib` on newer Python setups early prevented debugging delays.
- **Hardware Connection Awareness:** Identifying and addressing simple emulator targeting loops (`10.0.2.2`) smoothly.

## Key Insights & Learnings
- **Background Actions reduce blocking times for common triggers** (like email confirmation or triggers in Story 2.4).
- **Consolidating common providers (like `dio_provider`) early** is a lifesaver across layered stories (Story 2.5).
