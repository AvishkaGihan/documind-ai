## Successes & Strengths
- **Reuse-First Backend Orchestration:** Reusing `DocumentService.delete_document_for_user()` inside `UserService.delete_account()` delivered extremely fast velocity and guaranteed error safety across external cascades (Storage/Vectors).
- **Graceful Destruction Safeguards:** Implemented clear Material `AlertDialog` wrappers using `StatefulBuilder` directly in the view layer to isolate inline progress handling without inflating the main controller state footprint.
- **Dynamic Session Theming:** Centralized `themeModeProvider` successfully supports live Dark/Light recalculations leveraging existing inherited definitions flawlessly.
- **One-Shot Flash Messages Response:** Designing `auth_flash_message_provider.dart` avoided anti-patterns binding ephemeral success SnackBars directly into `go_router` route triggers.

## Challenges & Growth Areas
- **Theme Preference Persistence Range:** Keeping dark-mode choices restricted to active sessions leaves a feature gap upon app cold restarts. Needs local cache bindings.
- **AlertDialog Frame Timers in Tests:** Asserting destructive dialog pops can trigger frame-render race conditions if not using precise bounded `pump()` intervals.

## Key Insights & Learnings
- **Isolated Flash Providers:** Using Riverpod `StateProvider` is highly superior to passing success flags via route parameters or global singletons for stateful redirects.
- **Explicit Provider Bindings:** Passing `ref.read(authApiProvider)` directly rather than injecting full widget scopes maintains cleaner SRP (Single Responsibility Principle) layouts.
